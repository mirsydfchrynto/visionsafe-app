1. SUMBER MODEL & TEKNOLOGI AI (The "Brain")
--------------------------------------------------------------------------------
Aplikasi VisionSafe tidak melatih (training) model dari nol karena itu membutuhkan
jutaan data gambar dan waktu berbulan-bulan. Kita menggunakan pendekatan 
"Applied AI" (AI Terapan).

- SUMBER MODEL: MediaPipe Face Landmarker (Dikembangkan oleh Google Research).
- JENIS MODEL: Deep Learning Convolutional Neural Network (CNN).
- FILE MODEL: `face_landmarker.task` (Ada di folder assets android).
- KEUNGGULAN: Model ini mampu mendeteksi 478 titik koordinat wajah (Landmarks) 
  secara 3D dalam waktu kurang dari 20 milidetik (Real-time).

--------------------------------------------------------------------------------
2. BAGAIMANA FITUR UTAMA "PENGAWAS MATA" BEKERJA?
--------------------------------------------------------------------------------
Proses ini terjadi di folder: `android/app/src/main/kotlin/.../VisionAnalyzer.kt`

Langkah-langkahnya (Workflow):
1. PENGAMBILAN GAMBAR: Kamera depan mengambil gambar (frame) setiap 1 detik.
2. DETEKSI LANDMARK: AI mencari titik mata kiri (ID: 33) dan mata kanan (ID: 263).
3. EKSTRAKSI KOORDINAT: AI mendapatkan nilai X (lebar), Y (tinggi), dan Z (kedalaman).
4. PERHITUNGAN MATEMATIKA (The Magic): Di sinilah Kecerdasan Terapan bekerja.

--------------------------------------------------------------------------------
3. RUMUS PERHITUNGAN JARAK (TRIANGLE SIMILARITY)
--------------------------------------------------------------------------------
Dosen sering bertanya: "Gimana HP tahu jarak mata padahal kameranya cuma satu?"
Jawabannya adalah GEOMETRI.

A. Menghitung Jarak Mata di Layar (Pixel IPD):
   Kita menggunakan rumus Pythagoras 3D (Euclidean Distance):
   Jarak_Pixel = Akar( (x2-x1)² + (y2-y1)² + (z2-z1)² )
   *Kenapa pakai Z? Agar jika wajah Master miring, perhitungannya tetap akurat.

B. Konversi Pixel ke Centimeter (Jarak Nyata):
   Kita menggunakan prinsip Kamera Lubang Jarum (Pinhole Camera Model).
   Rumus: Jarak_Asli = (Lebar_Mata_Asli * Focal_Length) / Jarak_Pixel
   
   - Lebar_Mata_Asli (IPD): Rata-rata manusia adalah 6.3 cm.
   - Focal_Length: Angka konstanta (820.0) yang didapat dari spesifikasi sensor kamera.
   - Jarak_Pixel: Hasil deteksi AI tadi.

CONTOH:
Jika mata di layar terlihat LEBAR (Pixel Besar), maka hasil pembagiannya KECIL.
Artinya: Mata DEKAT dengan layar.
Jika mata di layar terlihat SEMPIT (Pixel Kecil), maka hasil pembagiannya BESAR.
Artinya: Mata JAUH dari layar.

--------------------------------------------------------------------------------
4. TEKNIK SMOOTHING (LOW-PASS FILTER)
--------------------------------------------------------------------------------
Pernah lihat angka sensor yang lompat-lompat? Di VisionSafe, kita pakai filter:
Rumus: Jarak_Baru = (Jarak_Sekarang * 0.3) + (Jarak_Lama * 0.7)

Ini membuat pergerakan angka di UI menjadi sangat halus dan profesional, tidak
terpengaruh oleh goyangan tangan yang kecil (jitter).

--------------------------------------------------------------------------------
5. LOGIKA BLUR & EMERGENCY LOCK (Decision Support System)
--------------------------------------------------------------------------------
Setelah AI mendapatkan angka jarak (misal: 25 cm), sistem mengambil keputusan:

- JIKA Jarak < 35 cm: 
  Sistem mulai menghitung durasi pelanggaran. Jika bertahan lebih dari 1.5 detik, 
  maka `BlurOverlayManager` diperintahkan untuk memunculkan efek blur di layar.
  
- JIKA Pelanggaran > 10 detik: 
  Sistem masuk ke mode EMERGENCY LOCK. Layar menjadi sangat blur dan muncul 
  pesan peringatan keras. Ini untuk memaksa pengguna menjauhkan HP.

--------------------------------------------------------------------------------
6. IMPLEMENTASI WEB SERVICE (SUPABASE CLOUD)
--------------------------------------------------------------------------------
Aplikasi VisionSafe menggunakan arsitektur Cloud-First yang modern dengan 
Supabase sebagai backend utama:

- BACKEND: Supabase (PostgreSQL Engine).
- DATABASE: PostgreSQL dengan Row Level Security (RLS) untuk keamanan data.
- AUTH: Supabase Auth (Email & Google OAuth).
- FLOW: 
  Flutter -> Supabase SDK -> Batch Insert ke Tabel `telemetry`.
- AUTOMATION: Database Triggers (PL/pgSQL) digunakan untuk menghitung XP, 
  Level, dan durasi fokus secara otomatis di sisi server setiap kali data 
  telemetri masuk.

KEUNGGULAN ARSITEKTUR INI:
1. DATA INTEGRITY: Semua perhitungan statistik (XP/Level) dilakukan di server, 
   mencegah manipulasi data di sisi aplikasi.
2. SECURITY: Row Level Security memastikan user HANYA bisa membaca dan menulis 
   data mereka sendiri.
3. REAL-TIME: Perubahan XP dan Level bisa langsung didengar oleh aplikasi 
   mengalui Realtime Channels.
4. SCALABILITY: Database PostgreSQL siap menangani jutaan data telemetri.
